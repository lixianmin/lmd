# memory.md

AI agent entry point for the LMD project. Trigger keywords + file pointers only.

## Key Principles

1. **Standard library first**: Use Go standard library whenever possible. Only introduce external dependencies when the standard library cannot meet the requirement.

2. **Test-first (TDD)**: Always write unit tests before implementation code. This is a hard rule, not a suggestion.

3. **CLI-first**: All features must be accessible via CLI. The code library (`pkg/`) is a thin wrapper around the service layer.

## Project Structure

```
cmd/lmd/             - CLI entry point
internal/cli/        - Cobra command definitions
internal/dao/        - Database access layer (global dao.DB)
internal/service/    - Business logic layer
internal/tokenizer/  - Text segmentation (gse)
internal/embedding/  - Vector embedding (Ollama > llama-server fallback)
internal/chunker/    - Markdown-aware chunking (base64 stripped, code blocks split)
internal/formatter/  - Output formatting (text/json/md/csv)
internal/mcp/        - MCP server (stdio JSON-RPC)
pkg/                 - Public API for external Go projects
```

## Key Technical Decisions

- **Tokenizer**: go-ego/gse — `seg.SkipLog = true` suppress log noise
- **FTS**: gse pre-tokenize + FTS5 (unicode61 + porter)
- **Vector storage**: sqlite-vec (cosine, 1024 dim)
- **Embedding model**: `qwen3-embedding:0.6b-q8_0` via Ollama (优先); llama-server GGUF fallback
- **Embedding batch**: 8 chunks/batch, rune 级 800 字符截断; Ollama keep_alive=30m, warmup on init
- **Query embedding**: NO Instruct prefix — documents and queries embedded as-is
- **Fusion**: 加权线性组合 (0.7×向量 + 0.3×BM25)，按 ChunkId 合并分数，保留同一文档的多个 chunk（喂给 LLM）
- **DB indexes**: `documents(collection, path)`, `chunks(doc_id)` — see `internal/dao/schema.go`
- **Timezone**: GMT+8 (CST)
- **SQLite**: WAL mode, no migration system, `CreateTables()` in `schema.go`
