# memory.md

AI agent entry point for the LMD project. Trigger keywords + file pointers only.

## Key Principles

1. **Standard library first**: Use Go standard library whenever possible. Only introduce external dependencies when the standard library cannot meet the requirement.

2. **Test-first (TDD)**: Always write unit tests before implementation code. This is a hard rule, not a suggestion.

3. **CLI-first**: All features must be accessible via CLI. The daemon HTTP API is the primary interface (no public Go library API).

## Key Technical Decisions

- **Tokenizer**: go-ego/gse — `seg.SkipLog = true` suppress log noise
- **FTS**: gse pre-tokenize + FTS5 (unicode61 + porter)，默认策略 pos-or（POS 过滤后 OR 连接，过滤掉虚词/停用词/代词，保留名词+动词+形容词）
- **Vector storage**: sqlite-vec (cosine, 1024 dim)
- **Embedding model**: `batiai/qwen3-embedding:0.6b` via SiliconFlow API
- **LLM providers**: Pluggable (`internal/llm/`), Ollama (local) + SiliconFlow (cloud). Provider config orthogonal to function config (providers section has URL/APIKey, embedding/summary sections reference provider+model)
- **Summary system**: File-level summaries in `@summaries` collection, reuse chunks/chunks_fts/chunks_vec tables, `source_doc_id` links summary→original. Two-level search: Level 1 searches summaries, Level 2 searches within matched file chunks
- **Summary generation**: LLM auto-generated on dirty docs (hash change detection), cooldown period between cycles, `UpsertSummaryDoc` DAO with transaction
- **Embedding batch**: 8 chunks/batch, rune 级 800 字符截断（config.Embedding.Truncation，对齐 spec）
- **Chunker**: Markdown-aware + PlainText chunking, sliding window, **300 rune target**（从 1200 缩小，减少语义稀释），句级 overlap ~15%。.txt 用 PlainTextChunker（无标题/代码块感知），.md 用 MarkdownChunker
- **Multi-ext glob**: `expandGlobPattern()` 拆分 `{md,txt}` → 多个子 pattern，文件匹配任一即命中。默认 `**/*.{md,txt}`
- **Query embedding**: query prefix 从 EmbeddingConfig.QueryPrefix 读取（默认值与 qwen3-embedding 官方一致），不再是全局常量
- **Document embedding**: 原文直接 embedding，不加前缀
- **Vector query expansion**: 已移除 Rocchio PRF（在混合系统中增益递减，详见 `docs/superpowers/specs/2026-04-21-hybrid-search-lessons.md`）
- **Search granularity**: Chunk-level (FTS on chunks_fts, vector on chunks_vec, results are individual chunks)
- **Score normalization**: Vector: 1-distance (BM25 raw scores pass through)
- **Fusion**: RRF (k=60, 2x 权重 for primary lists, top-rank bonus)，按 ChunkId 合并分数。最终分数归一化到 [0, 1]（score/maxScore）
- **HyDE**: HyDE query expansion via SiliconFlow API (Qwen/Qwen3.5-9B)，OpenAI-compatible，按需调用
- **检索质量优化 spec**: `docs/superpowers/specs/2026-04-21-retrieval-quality-optimization.md` — 分块参数、指令前缀的讨论记录
- **Hybrid search 经验**: `docs/superpowers/specs/2026-04-21-hybrid-search-lessons.md` — 为什么 PRF+MMR 叠加效果差，业界最佳实践
- **Memory layer**: 记忆已合并到 documents+chunks 表，使用 `@episodic`/@knowledge collection 区分（`@` 前缀为系统保留）。memory/add、delete、update 接口保留，查询统一使用 /query。暂不实现时间衰减。旧表 memories/memories_fts/memories_vec 已移除，迁移逻辑在 `dao/db_init.go:migrateMemories()`
- **DB indexes**: `documents(collection, path)`, `chunks(doc_id)` — see `internal/dao/db_init.go`
- **Timezone**: GMT+8 (CST)
- **SQLite**: WAL mode, no migration system, `CreateTables()` in `db_init.go`
- **syncIndex 并发**: syncIndex 持有 rebuildMu，避免与 embedChunks/embedMemories/handleRebuild 竞争
- **EnsureDaemon**: 30 秒超时，避免无限等待
- **POS-based query策略**: 默认 pos-or（GSE 词性标注 + 英文 Brown Corpus 映射 → 过滤实词 → OR 连接），`--strategy` 支持 or, df, pos-or, pos-must, pos-weight, and
- **English POS map**: `internal/tokenizer/en_pos.go`，Brown Corpus (NLTK)，46,066 词 → n/v/adj/adv 四类标签，由 `gse.go` 在 POS 标注时查表覆盖 GSE 的 "x" 标记

## 经验教训

- 记忆存储模型：1 条记忆 = 1 行 document + 1 个 chunk，复用 chunks_fts + chunks_vec 索引。collection 使用 `@` 前缀标记系统保留类型（如 `@episodic`）
- 记忆查询统一走 /query 接口（而非独立 memory/query），LLM 不区分来源
- chunks_fts 是 content-sync FTS5 表（`content='chunks'`），INSERT 自动同步，无需额外显式插入

- UTF-8 文本截断必须用 rune 级（`[]rune(s)[:n]`），不能按 byte 截断，中文字符 3 字节会切断
- Scan() 错误必须检查，尤其是循环中的 Scan — 未检查的 Scan 会用零值，可能导致删除错误数据
- config 默认值必须与最新 spec 一致（truncation 曾是 300，spec 已改为 800）
- DB schema DEFAULT 值必须与应用默认值同步（glob_pattern 曾是 `**/*.md`，需与 CLI 默认 `**/*.{md,txt}` 一致）
- **67 个声称的 bug 修复审计结果**：仅 12 个是真正的 bug (18%)，19 个合理改进 (28%)，14 个过度修改 (21%)，2 个有害 (3%)。审计方法论见本 session 对话记录。5 个关键 bug 已有回归测试覆盖（RenameCollection 文档更新、DeleteDocument 孤儿清理、CSV CJK 截断、PlainTextChunker 行号、ignorePatterns 排除文件）
- **禁止 review-fix 循环**：不允许连续多轮 "发现新 bug → 修复" 的模式。每轮修复会制造需要下一轮修复的问题（如 R1 移除 stmt.Close 但没移除 Prepare，制造了 statement 泄漏，R2 才真正修复）。所有修改必须先有失败测试
- **Go `omitempty` + `bool` = 陷阱**：`json:"think,omitempty"` 配合 `Think: false` 时，`false` 被视为零值而省略该字段。Ollama qwen3.5 在缺少 `"think": false` 时默认开启 thinking mode，导致 KV cache 爆炸、模型崩溃。**对 API 的 bool 控制字段，永远不要用 `omitempty`**。
- **llama.cpp 已归档**：`archive/llamacpp` 分支 + `v0.1-llamacpp` tag。所有 llama.cpp CGo 代码已移除，embedding 改用 SiliconFlow API，LLM 改用 Ollama/SiliconFlow
- **Ollama API `think` 参数**：qwen3 系列 `think` 是 top-level 请求参数，不是 `options` 内的 `enable_thinking`。后者会报 `"invalid option provided"` 并导致挂起
- [AGENTS.md 建议] 禁止连续多轮 review-fix 循环。任何代码修改必须先有失败测试用例证明 bug 存在，然后才能写修复。单次 session 最多允许 1 轮 review+fix
- [AGENTS.md 建议] Go 代码中使用 `any` 代替 `interface{}`
