# memory.md

AI agent entry point for the LMD project. Trigger keywords + file pointers only.

## Key Principles

1. **Standard library first**: Use Go standard library whenever possible. Only introduce external dependencies when the standard library cannot meet the requirement.

2. **Test-first (TDD)**: Always write unit tests before implementation code. This is a hard rule, not a suggestion.

3. **CLI-first**: All features must be accessible via CLI. The daemon HTTP API is the primary interface (no public Go library API).

## Key Technical Decisions

- **Tokenizer**: go-ego/gse — `seg.SkipLog = true` suppress log noise
- **FTS**: gse pre-tokenize + FTS5 (unicode61 + porter)
- **Vector storage**: sqlite-vec (cosine, 1024 dim)
- **Embedding model**: `Qwen3-Embedding-0.6B-Q8_0.gguf` via llama-go (CGo 内嵌，daemon 进程内)
- **Embedding architecture**: tcpipuk/llama-go CGo bindings，按需加载 + 10min 空闲释放，Metal GPU 加速
- **Embedding batch**: 8 chunks/batch, rune 级 300 字符截断（与 chunkSize 对齐）
- **Chunker**: Markdown-aware chunking, sliding window, **300 rune target**（从 1200 缩小，减少语义稀释），句级 overlap ~15%
- **Query embedding**: `Instruct: Given a web search query, retrieve relevant passages that answer the query\nQuery: {query}`（Qwen3-Embedding 官方要求）
- **Document embedding**: 原文直接 embedding，不加前缀
- **Vector query expansion**: Rocchio PRF — BM25 top3 的已有 embedding 加权平均，α=0.6 β=0.4，零额外模型调用。BM25 无结果时回退到普通向量搜索
- **Search granularity**: Chunk-level (FTS on chunks_fts, vector on chunks_vec, results are individual chunks)
- **Score normalization**: Vector: 1-distance (BM25 raw scores pass through)
- **Fusion**: RRF (k=60, 2x 权重 for primary lists, top-rank bonus)，按 ChunkId 合并分数，保留同一文档的多个 chunk（喂给 LLM）。最终分数用 `1/rank`（参考 QMD），范围 (0, 1]
- **MMR**: RRF fusion 后用 MMR 去重排序，λ=0.7，偏向相关性
- **HyDE**: HyDE query expansion via SiliconFlow API (Qwen/Qwen3.5-9B)，OpenAI-compatible，按需调用
- **检索质量优化 spec**: `docs/superpowers/specs/2026-04-21-retrieval-quality-optimization.md` — 分块参数、指令前缀、Rocchio PRF、MMR 的完整讨论记录
- **Memory layer**: Agent memories: independent table, fact/episode/relation, exponential time-decay scoring. Add 只插内容，embedding 由 daemon embedTicker 后台批量处理（5s 间隔）
- **DB indexes**: `documents(collection, path)`, `chunks(doc_id)` — see `internal/dao/schema.go`
- **Timezone**: GMT+8 (CST)
- **SQLite**: WAL mode, no migration system, `CreateTables()` in `schema.go`

## Build & Test

```bash

make install                  # Install to GOPATH/bin, auto-stops old daemon
make test                     # Run all tests (tags: fts5)
make vet                      # Static analysis
make lint                     # vet + fmt
make e2e                      # End-to-end smoke test
make integration-basic        # CLI integration tests
lmd rebuild                   # Full rebuild (keeps collections)
```
